##
## JPEG FUNCTION-PARALLEL decoder (2c) - full 5-stage pipeline.
##
##   tile 0 : VLD    (p1, ALONE - needs its private DMA region)
##   tile 1 : IDCT (p1) + IQZZ (p2) + CC (p3)  - compute-only, no DMA regions
##   tile 2 : raster (p2, ALONE - needs its private DMA region)
##
## PLATFORM CONSTRAINT (found the hard way): every partition automatically gets
## an MMU window to every declared partition-private region. A window into a
## SAME-TILE sibling's region hits "overlaps locked memory region" at load time
## (own region and cross-tile regions are fine). Additionally the DMA library
## only accepts local buffers inside declared private regions. Hence the two
## DMA actors (VLD source, raster sink) must be ALONE on their tiles, and the
## three pure-compute actors share tile 1 (which also satisfies the
## multiple-partitions-per-tile requirement).
##
## Stages communicate via FIFOs in the vep-wide shared memory (memshared0).
##
## memory: system takes the first 32 KiB of each tile; a partition's start
## offset must be a MULTIPLE of its size.
##
on tile 0 partition 1 has 8KiB stack in 64KiB memory starting at 64KiB
on tile 1 partition 1 has 4KiB stack in 32KiB memory starting at 32KiB
on tile 1 partition 2 has 8KiB stack in 32KiB memory starting at 64KiB
on tile 2 partition 2 has 8KiB stack in 32KiB memory starting at 64KiB
##
## private DMA staging regions (1 KiB-aligned buffers), DMA endpoints only
##
on tile 0 partition 1 shares 3KiB private memory
on tile 2 partition 2 shares 3KiB private memory
##
## pipeline FIFOs in the vep-wide shared memory (reachable by all tiles);
## 4 FIFOs x depth 2 + admin ~= 21 KiB. Top 32 KiB (96..128 KiB) is known-good.
##
in shared memory memshared0 shares 32KiB private memory starting at 96KiB
##
## schedule. All slot tables are 67000 cycles. System uses 0-4999, slots are
## separated by 2000-cycle VKERNEL slots.
##
## Tile 1 split (F-02 rebalance, tuned on cat-big measurements): the original
## 40000/7000/7000 split starved IQZZ (~25 revolutions/MCU paced the whole
## pipeline; "IDCT-heavy" was a myth from average-ET numbers that ignored the
## token-copy cost). With the S-02 copy trim the MEASURED per-firing work is
## IQZZ 103.6k/52.5k, IDCT 72.9k/37.1k, CC 46.8k/14.6k cycles (16x16 / 8x8
## MCUs). Steady-state pace per stage = work * 67000/slot (validated within 1%
## on cat and cat-big), so the split is sized to the GRADED 8x8 images:
## IQZZ 27000 / IDCT 19000 / CC 8000 -> cat-big pace ~131k cyc/MCU ~= 85 s
## (+ ~4.5 s gather) < the 100 s self_check timeout. Slots in DATAFLOW order
## IQZZ -> IDCT -> CC so one MCU traverses all three stages per revolution.
## (5000 + 2000 + 27000 + 2000 + 19000 + 2000 + 8000 + 2000 = 67000).
##
on tile 0 partition 1 has 58000 cycles of 67000 starting at  7000
on tile 1 partition 1 has 46000 cycles of 67000 starting at  7000
on tile 1 partition 2 has 10000 cycles of 67000 starting at 55000
on tile 2 partition 2 has 58000 cycles of 67000 starting at  7000
